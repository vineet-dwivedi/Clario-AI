# Auth Flow Notes

## Frontend

- `useAuth.js` is the main bridge between the UI and the backend auth API.
- Login sends `{ email, password }`.
- Register sends `{ username, email, password }`.
- Register does not auto-login because the backend requires email verification first.

## Backend Contract

- `POST /api/auth/register` creates the user and sends a verification email.
- `POST /api/auth/login` logs the user in only after email verification.
- `GET /api/auth/get-me` reads the logged-in user from the auth cookie.

## UI Behavior

- `Login.jsx` shows backend success/error messages directly in the auth card.
- `Register.jsx` validates password confirmation and terms checkbox before calling the hook.
- On successful registration, the user is redirected to `/login` with a message to verify email first.
